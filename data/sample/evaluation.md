# Оценка качества RAG

## Метрики поиска
Recall@k показывает, попал ли нужный документ в первые k результатов. MRR
(Mean Reciprocal Rank) усредняет величину, обратную позиции первого верного
ответа, и поэтому чувствителен к тому, насколько высоко стоит правильный
документ. nDCG дополнительно учитывает градации релевантности и затухание
по позиции.

## Метрики ответа
Faithfulness измеряет, насколько ответ опирается на найденный контекст, без
выдумок. Answer relevance измеряет, насколько ответ отвечает на сам вопрос.
Эти две метрики ортогональны: ответ может быть верным по фактам, но не по теме.

## RAGAS и LLM-judge
RAGAS — библиотека автоматической оценки RAG, считающая faithfulness,
context precision и context recall. LLM-judge использует сильную модель как
арбитра, оценивающего ответ по шкале. Оба подхода точнее лексических прокси,
но дороже и сами могут ошибаться.

## Золотой набор
Для воспроизводимой оценки нужен золотой набор: вопросы с заранее размеченным
правильным источником. Прогон по нему даёт сравнимые во времени числа.
